## IMPORTANT WARNING

The data generated by this tool is completely synthetic.  
It is based on mechanistic models, simplifications, and calibrations performed specifically for training machine learning models.

**DO NOT USE THIS DATA FOR CLINICAL DIAGNOSES, TREATMENT, OR ANY OTHER MEDICAL DECISIONS.**  
This tool is intended exclusively for educational and research purposes in the field of ML.

---

## About the Project

`SyntheticHealthSimulator` generates realistic (from a statistical patterns perspective) panel data about the health status of a virtual population over 20 years. Each "patient" has:

- Demographic characteristics (age, sex, height);
- Genetic profile (polygenic risks);
- Lifestyle history (nutrition, physical activity, smoking, sleep, stress);
- Annually measured biomarkers (weight, blood pressure, cholesterol, HbA1c).

Based on this data, risks of developing various diseases (cardiovascular disease, type 2 diabetes, stroke, NAFLD, colorectal cancer, cirrhosis) are calculated and outcomes are modeled.

The main purpose is to create high-quality datasets for training prediction models, investigating the influence of lifestyle factors, and testing algorithms for handling missing data.

## Usage

Basic data generation (5000 people, 20 years of observation):

```bash
python generator.py
```

All settings are defined in the `main()` function:

- `N_PEOPLE` — number of synthetic patients.
- `YEARS` — observation period (years).
- `seed` — random seed for reproducibility.
- `diabetes_threshold` — diabetes risk calibration threshold (default 18.7, provides ~15% prevalence).

After completion, five CSV files (and corresponding JSON metadata files) will be created in the `data/synthetic` folder.

## Dataset Validation

The repository includes a built‑in validator (`validator.py`) to ensure that the generated data meets the expected statistical properties, physiological ranges, and correlations defined in the [scientific reference](SCIENTIFIC_REFERENCE.md). The validator checks:

- File completeness and structure
- Demographic distributions (age, sex, height)
- Genetic risk distributions
- Biomarker ranges and hard clipping
- Temporal trends (e.g., HDL decrease, HbA1c increase)
- Disease prevalence and risk‑outcome correlations
- Missing‑at‑random (MAR) patterns
- Cross‑dataset consistency (person IDs, BMI)

To validate a locally generated dataset:

```bash
python validator.py --local data/synthetic_v1.0.2
```

You can also validate a dataset directly from a GitHub release:

```bash
python validator.py --github kavalex/SyntheticHealthSimulator --version v1.0.2
```

An optional `--report report.json` flag saves a detailed JSON report.

If all checks pass, you will see:

```
ALL CHECKS PASSED SUCCESSFULLY!
```

Otherwise, warnings or errors will be displayed with details.

## Structure of Generated Data

All files except the aggregated one have a panel structure: one row or set of indicators is stored for each patient and year of observation. Below are the columns for each file.

### 1. Baseline Cohort (`01_cohort_baseline.csv`)
One row per patient. Contains immutable characteristics and initial values.

- **Identifier and demographics**: `person_id`, `age_start`, `sex`, `height_cm`
- **Anthropometry**: `weight_start_kg`, `bmi_start`, `muscle_mass_start`
- **Energy metabolism**: `bmr_start_kcal`, `base_calories`
- **Genetic risks**: `genetic_risk_cvd`, `genetic_risk_diabetes`, `genetic_risk_stroke`, `genetic_risk_nafld`, `genetic_risk_colorectal`, `genetic_risk_cirrhosis`
- **Initial biomarkers**: `initial_hdl_mgdl`, `initial_total_cholesterol_mgdl`, `initial_sbp_mmhg`, `initial_hba1c_percent`

### 2. Lifestyle History (`02_lifestyle_history.csv`)
Lifestyle indicators and calculated parameters are recorded for each patient and each year.

- **Identifier, year, age**: `person_id`, `year`, `age`
- **Physical indicators (true values, without noise)**: `weight_kg_true`, `muscle_mass_factor_true`, `bmr_kcal_true`, `tdee_kcal_true`
- **Nutrition**: `protein_pct`, `fat_pct`, `carb_pct`, `saturated_fat_pct`, `simple_carbs_pct`, `fiber_g_day`, `sodium_mg_day`, `macro_imbalance`, `simple_carbs_g`
- **Lifestyle**: `alcohol_g_per_week`, `cardio_met_minutes`, `strength_met_minutes`, `total_met_minutes`, `cigarettes_per_day`, `stress_level`, `sleep_hours`, `cumulative_smoking`

### 3. Biomarkers (`03_biomarkers_history.csv`)
Annual biomarker values (with added noise).

- **Identifier, year**: `person_id`, `year`
- **Biomarkers**: `weight_kg`, `hdl_mgdl`, `total_cholesterol_mgdl`, `sbp_mmhg`, `hba1c_percent`
- **Derived indicators**: `bmi`, `bmi_corrected`, `non_hdl_mgdl`, `non_hdl_mmol`, `muscle_mass_factor`

### 4. Risks and Outcomes (`04_health_risks.csv`)
Calculated 10-year risks and binary flags for disease presence over the entire observation period.

- **Identifier**: `person_id`
- **Risks**: `cvd_risk_10year`, `diabetes_risk_10year`, `stroke_risk_10year`, `nafld_risk_10year`, `colorectal_risk_10year`, `cirrhosis_risk_10year`
- **Outcomes**: `has_cvd`, `has_diabetes`, `has_stroke`, `has_nafld`, `has_colorectal`, `has_cirrhosis`
- **Additional**: `health_score`, `primary_death_cause`, `estimated_event_age`, `sex`

### 5. Aggregated Dataset with Missing Values (`05_aggregated_dataset_with_missing.csv`)
One row per patient. Contains averaged and start/end values of lifestyle factors, final biomarkers, risks, outcomes, and synthesized missing values (via MAR mechanism). Missing values may appear in columns: `final_sbp_mmhg`, `final_hdl_mgdl`, `final_total_cholesterol_mgdl`, `final_hba1c_percent`, `avg_alcohol_g_per_week`, `avg_total_met_minutes`.

- **Identifier and demographics**: `person_id`, `sex`, `age_start`, `age_end`
- **Averaged lifestyle factors**: `avg_alcohol_g_per_week`, `avg_total_met_minutes`, `avg_sleep_hours`, `avg_stress_level`, `avg_fiber_g_day`, `avg_sodium_mg_day`, `avg_saturated_fat_pct`, `avg_simple_carbs_g`, `avg_macro_imbalance`
- **Weight and muscle mass dynamics**: `bmi_start`, `bmi_end`, `bmi_change`, `weight_start_kg`, `weight_end_kg`, `weight_change`, `muscle_mass_factor_start`, `muscle_mass_factor_end`, `muscle_mass_factor_change`
- **Final biomarkers**: `final_hdl_mgdl`, `final_total_cholesterol_mgdl`, `final_sbp_mmhg`, `final_hba1c_percent`, `final_non_hdl_mmol`
- **Smoking**: `avg_cigarettes_per_day`, `cumulative_smoking_end`
- **Genetic risks**: all 6 (`genetic_risk_cvd`, `genetic_risk_diabetes`, `genetic_risk_stroke`, `genetic_risk_nafld`, `genetic_risk_colorectal`, `genetic_risk_cirrhosis`)
- **Risks and outcomes**: all 6 risks and outcomes (as in file 4)
- **Additional**: `health_score`, `primary_death_cause`, `estimated_event_age`

## Calibration and Configuration

All coefficients in the model are intentionally reduced compared to epidemiological data to prevent dominance of a single factor and ensure balanced ML model training. The key configurable parameter:

- **Diabetes threshold** (`diabetes_threshold`) — regulates diabetes prevalence. Value 18.7 gives ~15% incidence over 20 years.

Other coefficients (SCORE2 intercept, smoking, alcohol, sodium effects) are described in detail in the [Scientific Reference](SCIENTIFIC_REFERENCE.md).
